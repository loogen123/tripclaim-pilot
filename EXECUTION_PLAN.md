# 交通费自动审批项目执行计划

## 已完成
- 第一版自动审批引擎已可运行：文件夹扫描、材料分类、规则校验、结果导出（json/md）
- 已建立基础规格文件：spec、tasks、checklist

## 当前目标
- 在自动审批基础上，增加“前端/客户端人工复核审批台”
- 保留审批留痕，支持复核改判

## 里程碑计划
### 里程碑 1：规则校准
- 使用 `报销材料`（已通过样本）作为基准样本
- 调整规则，确保通过样本稳定输出“通过”
- 固化规则配置文件，支持后续维护

### 里程碑 2：后端 API
- `POST /cases` 创建案例（输入文件夹路径）
- `POST /cases/{id}/run` 执行自动审批
- `GET /cases/{id}` 查询审批结果
- `POST /cases/{id}/manual-review` 人工改判并留痕

### 里程碑 3：审批前端
- 页面提供案例创建、自动审批触发、结果查看
- 支持人工复核（通过/驳回/转人工）与备注提交

### 里程碑 4：持久化
- SQLite 存储案例、问题明细、人工复核记录
- 支持后续审计追踪与复盘

### 里程碑 5：验收
- 通过样本应输出“通过”
- 缺材料样本应输出“驳回/转人工”
- 前端可完成“自动审批 -> 人工复核 -> 留痕”闭环
