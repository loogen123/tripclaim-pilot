# 票据防伪与校验机制阶段性计划 (Plan)

本项目遵循“极致省钱”与“开源优先”的原则，分阶段推进防造假机制的落地。

## 里程碑 1：核心基线与强拦截机制（第 1 周 - 最低可用省钱版）
**目标：建立防伪基础结构，拦截最常见的造假手段，实现 0 到 1 的突破。**

1.  **重构数据模型**：在 `models.py` 中扩充 `Document` 的属性，支持存放结构化字段、`fraud_score`、`verify_status` 和 `fraud_reasons`。
2.  **结构化字段提取**：在 `engine.py` 中分类之后，使用正则或轻量级方案，针对 `transport_invoice` 提取票号、金额、日期等用于查验的必备字段。
3.  **联网查验集成**：
    - 新增 `services/verification.py` 模块，封装对税务局或第三方（如航信、百望云）的 API 调用逻辑。
    - 在规则引擎执行前，异步调用此服务，将结果回填至 `doc.fields` 和 `verify_status`。
4.  **强规则阻断**：修改 `rules.py`，新增最高优先级规则：若 `verify_status == "invalid"`，直接触发 `high` 级别 Issue 驳回。
5.  **重复票据拦截**：基于本地历史数据库，对发票号或行程单号建立唯一索引，命中重复票据直接阻断。

## 里程碑 2：图像本地校验与逻辑一致性（第 2 周）
**目标：无需云端调用的本地防伪能力，重点打击篡改与逻辑错误。**

1.  **字段逻辑一致性校验**：
    - 开发本地脚本，交叉校验：大小写金额是否匹配、税额计算是否正确（金额 × 税率 = 税额）。
    - 校验差旅行程日期与发票日期的合理性（排除提前很久开票或行程冲突的情况）。
2.  **图像哈希去重**：
    - 引入 `ImageHash` (如 `pHash`) 算法。
    - 对所有传入的票据图片生成指纹并比对，拦截不同截图、轻微修改的重复报销（针对无法提取出结构化票号的纸质票或截图）。
3.  **评分机制上线**：整合上述本地规则，输出基础的 `fraud_score`。

## 里程碑 3：AI 辅助识别与篡改检测（第 3-4 周 - 进阶版）
**目标：应对高级造假（如 PS 修改），提升识别兜底能力。**

1.  **篡改痕迹检测 (ELA/VLM)**：
    - 针对高额报销或疑似造假单据，调用图像篡改检测模型（如 Error Level Analysis）。
    - 或调用多模态大模型（如 GPT-4o）进行针对性视觉异常检查（字体、拼接、边缘残留）。
2.  **OCR 双通道交叉验证**：
    - 引入 `PaddleOCR` 或其他开源高精度 OCR 作为第二通道。
    - 当原 OCR 提取的金额、票号置信度低时，自动触发双通道识别并做 Diff，冲突时显著提高 `fraud_score`。
3.  **决策引擎调优**：
    - 在前端展示详细的 `fraud_reasons`。
    - 根据历史真实样本调优 `fraud_score` 阈值（自动拒绝线、人工复核线）。
